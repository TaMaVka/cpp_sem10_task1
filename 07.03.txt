ИСХОДНЫЙ КОД
------------------------------------------------------------------------------

void test(Person const& person)
{
    std::cout << "test: " << person.name() << '\n';
    if (person.grade() == 10 || person.salary() > 1'000'000)
    {
        save(Status::success, person.id());
    }
    else
    {
        save(Status::failure, person.id());
    }
}

Предположения:
- Person — пользовательский класс; типы, возвращаемые его методами
  (name(), grade(), salary(), id()), точно не известны и могут быть
  пользовательскими.
- Status — пользовательское перечисление с областью видимости (enum class).

НАЙДЕННЫЕ ЭЛЕМЕНТЫ ВЕТВЛЕНИЯ И ИСКЛЮЧЕНИЙ
-------------------------------------------------------------------------------

Элементы разделены на две категории:
[Ветвление] - нормальное ветвление потока управления (Normal Control Flow).
[Исключение] - скрытая или явная генерация исключений (Exceptional Control Flow).

1. [Исключение] Обращение к std::cout << " test : "
   Обоснование: Работа со стандартным потоком может сгенерировать std::ios_base::failure, если установлен флаг failbit/badbit, или std::bad_alloc при ошибке буферизации.
2. [Исключение] Вызов метода person.name()
   Обоснование: Тело пользовательского метода может содержать логику, генерирующую любые исключения (например, std::runtime_error).
3. [Исключение] Возврат результата name() по значению
   Обоснование: Создание временного объекта (например, std::string) может выбросить std::bad_alloc при выделении памяти.
4.[Исключение] Вызов operator<< для результата name()
   Обоснование: Перегруженный оператор вывода для пользовательского типа может бросить исключение при форматировании.
5. [Исключение] Вызов operator<< для символа '\n'
   Обоснование: Аналогично п.1, риск ошибки I/O потока.
6. [Исключение] Уничтожение временного объекта от name()
   Обоснование: Если деструктор возвращаемого типа не помечен как noexcept, его вызов в конце выражения может бросить исключение.

--- БЛОК УСЛОВИЯ (IF) ---
7.[Исключение] Вызов метода person.grade()
   Обоснование: Пользовательский метод может выбросить исключение.
8. [Исключение] Возврат результата grade()
   Обоснование: Копирование возвращаемого значения может бросить исключение (например, нехватка памяти).
9.[Исключение] Оценка выражения operator==
   Обоснование: Если оператор сравнения перегружен для пользовательского типа, он является обычным вызовом функции и может бросать исключения.
10.[Ветвление] Короткое замыкание (Short-circuit) оператора ||
    Обоснование: Нормальное ветвление. Если левая часть (grade() == 10) равна true, правая часть условия игнорируется.
11. [Исключение] Вызов метода person.salary()
    Обоснование: Выполняется только если не сработало короткое замыкание. Риск генерации исключения внутри метода.
12. [Исключение] Возврат результата salary()
    Обоснование: Копирование/создание временного объекта для возвращаемого значения.
13. [Исключение] Оценка выражения operator>
    Обоснование: Аналогично п.9, перегруженный оператор может бросить исключение.
14. [Исключение] Оценка выражения operator||
    Обоснование: Если логическое ИЛИ перегружено для пользовательских типов, оно теряет свойства короткого замыкания и может генерировать исключения.
15. [Исключение] Приведение результата к типу bool
    Обоснование: Условие оператора if требует типа bool. Если используется пользовательский operator bool(), он может бросить исключение.
16. [Исключение] Уничтожение временных объектов условия
    Обоснование: В конце условия if уничтожаются объекты от grade() и salary(). Их деструкторы могут выбросить исключения.
17. [Ветвление] Условный оператор if
    Обоснование: Явное ветвление потока управления на true (блок if) и false (блок else) на основе результата проверки.

--- БЛОК TRUE (ВЕТВЬ УСПЕХА) ---
18. [Исключение] Вызов метода person.id()
    Обоснование: Обращение к методу в ветке true может бросить исключение.
19. [Исключение] Возврат результата id() по значению
    Обоснование: Риск исключения при вызове копирующего конструктора.
20. [Исключение] Вызов функции save(Status::success, ...)
    Обоснование: Сама функция сохранения может выбросить исключение (ошибка записи, разрыв соединения и т.д.).
21. [Исключение] Уничтожение временных объектов ветви true
    Обоснование: Деструктор результата id() или временно сконвертированных аргументов может бросить исключение при выходе из блока.

--- БЛОК FALSE (ВЕТВЬ НЕУДАЧИ) ---
22. [Исключение] Вызов метода person.id()
    Обоснование: Обращение к методу в ветке false может бросить исключение.
23.[Исключение] Возврат результата id() по значению
    Обоснование: Риск исключения при копировании объекта.
24. [Исключение] Вызов функции save(Status::failure, ...)
    Обоснование: Аналогично п.20, генерация исключения внутри функции сохранения.
25.[Исключение] Уничтожение временных объектов ветви false
    Обоснование: Риск исключения в деструкторах очищаемых объектов (результата id() и аргументов) перед выходом из функции.
